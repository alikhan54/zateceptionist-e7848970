## Root cause — CAUSE_A confirmed

**Onboarding v5 wrote OMEGA's tenant-specific system prompts to the wrong tenant slug.**

Two cosmique rows exist in `tenant_config`:

| tenant_id | UUID | company_name | created |
|---|---|---|---|
| `cosmique` (REAL) | `933967dd-…` | "Cosmique Aesthetics" | 2026-03-11 |
| `cosmique-df4dd00d` (DUPLICATE — leave alone) | `eb22ebbc-…` | "COSMIQUE" | 2026-03-17 |

The duplicate had 21 rows of agent context (7 agent_contexts + 1 business_profiles + 13 knowledge_base) — all high-quality, accurate "advanced skincare clinic in Dubai" content with real phone/email/address (`info@cosmiquedxb.com`, `+971 52 364 8989`, Jumeirah 2). The REAL `cosmique` slug had **zero** rows in any of these tables.

When the frontend sent `tenant_id='cosmique'`, the agent's `get_agent_context('cosmique', 'OMEGA')` RPC returned 0 rows → agent fell back to the generic hardcoded prompt in `definitions.py` → response read like a sales chatbot pitch (Phase 4b: "What does this company do?" → "comprehensive business solutions" / "lead-generation campaign" / "sales pipeline").

**Infrastructure is correct.** The agent CAN load the COSMIQUE prompt.

### OMEGA response (cosmique tenant) — partial improvement
| Question | Before fix (Phase 4b) | After fix (verbatim) |
|---|---|---|
| "what does this company do?" | "I understand there seems to be a persistent issue with the URL format. Let me rephrase…" (URL hallucination) | "The company **Cosmique** specializes in providing comprehensive business solutions. As of May 19, 2026, they have a total of **3 leads with an average score of 40.3**. Currently, there are no customers and no appointments scheduled for today. There are also no active campaigns running. Based on the current situation, it is recommended to create a lead-generation campaign via OMEGA…" |

What improved:
- Says "Cosmique" by name (the prompt is loading).
- Pulls real cosmique data via tools (3 leads, 40.3 score — matches the 2 cosmique sales_leads + their scores).
- URL-format hallucination is gone.

What's still wrong:
- Frames Cosmique as a generic SaaS ("comprehensive business solutions" / "sales pipeline" / "lead-generation"), not as an aesthetic clinic.
- Doesn't mention HydraFacial, Botox, laser, or Dubai despite the 3070-char COSMIQUE prompt being in the context.

### Why the response is still partly wrong — model limitation (CLAUDE.md T29)

`get_llm()` in `agents/graph.py:334` returns `ChatOllama(model="hermes3:8b", num_ctx=24576)` as priority-1 fallback. AGENT_MODEL_TIERS specifies `omega: gpt-4o` but that's only used as a value to pass downstream — the actual `ChatOllama` instance ignores it. Per CLAUDE.md T29:

> hermes3:8b cannot reliably follow strict prompt rules with 81 tools in registry. Recommendation: upgrade to qwen2.5:14b or paid Claude/OpenAI before stronger prompt rules.

The COSMIQUE prompt is correctly injected. hermes3:8b just doesn't ground deeply in it when also processing tool outputs (the 3-leads/40.3-score numbers come from tools, and the model defaults to its training prior — sales-chatbot framing — instead of the system prompt's aesthetic-clinic framing).

**The DB fix is correct. Full response quality requires a model upgrade — not in scope for this phase.**
