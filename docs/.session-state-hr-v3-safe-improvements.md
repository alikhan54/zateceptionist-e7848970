## What changed

### Issue 1 — Job posting interview questions
- `src/pages/hr/Recruitment.tsx` — Post Job dialog (manual mode) gains an
  Interview Questions section: AI Generate button, manual Add, per-question
  type/skills/rubric editing, trash to remove.
- After job creation, `hr_job_requisitions.interview_questions` is patched
  with the array via Supabase from the client (the `/hr/job/ai-create`
  webhook doesn't pass interview_questions through today).
- AI generation uses new n8n workflow **420 HR Generate Job Questions v1.0**
  (`K46ve7frdaHYBpn2`, active). Backed by Ollama `hermes3:8b` via
  `host.docker.internal:11434`. Returns ~7 questions per call.

### Issue 2 — Company policy edit + version history
- `src/pages/hr/Documents.tsx` — gains "Edit Content" menu item on syncable
  doc rows (policy / handbook / contract / code_of_conduct / sop /
  guidelines). Edit dialog has rich Textarea editor + effective_date +
  optional re-sync-to-AI-agents checkbox.
- Each save creates a NEW row: `version_number = previous + 1`,
  `parent_document_id` points to original, `is_current_version = true` on
  new, `false` on previous. `edited_via = 'inline_edit'`.
- Optional sync hits existing `/hr/document/sync-to-agents` webhook.

### Issue 3 — Admin vs Staff panels
- New `src/pages/my/MyHR.tsx` — single-route page with 7 tabs
  (Profile / Attendance / Leaves / Payslips / Training / Documents / Reviews)
- URL pattern: `/my/:tab` — `/my/profile`, `/my/attendance`, etc.
  Default redirect `/my → /my/profile`.
- Employee record auto-resolved by `auth_user_id` first, then by
  `company_email` fallback. Friendly error card if no match found.
- `src/components/NavigationSidebar.tsx` — new `myHRSection` (collapsible,
  `staffOnly: true`) shown only when `authUser.role === 'staff'`. Admin and
  manager see the existing /hr/* admin routes.
- No route guards added — existing `canAccessSection` already gates /hr/*
  via `staff_permissions.can_access_hr`. /my/* is unrestricted by design
  (every authenticated employee can see their own data).

### Issue 4 — Real-time "Who's Working Now"
- New `src/components/hr/WhoIsWorkingNow.tsx` — Supabase realtime channel
  on `hr_attendance` + 30s react-query refetch. Shows avatars, name, role,
  live duration ("3h 12m"). Pulses green dot to signal live.
- Pinned to top of HR Dashboard `overview` tab AND admin `/hr/attendance`
  page header.
- New `src/components/hr/MyAttendanceWidget.tsx` — staff-facing punch
  in/out card on `/my/attendance`. Captures geolocation (optional),
  computes `work_hours` on punch-out, shows live duration counter.
- **Design choice:** "currently working" = `check_out_time IS NULL AND
  work_date = today`. We deliberately did NOT add an
  `is_currently_active` boolean — that would denormalize the same fact
  and drift under concurrent updates.

## Known limits

1. **Manual mode only** for interview questions. URL / Paste-Description
   modes don't capture interview_questions today — admin can edit them
   on the requisition after creation (TODO if the user wants this).
2. **Question generation** is Ollama-backed (free, local). If demand goes
   up, swap to OpenAI/Claude when the credentials are restored in this n8n
   instance — the workflow is structured to drop in either.
3. **My HR profile** is read-only. Salary/role edits require admin action.
   "Edit personal contact info" tab not built (deferred).
4. **Local browser test via `npm run preview`** was NOT executed in this
   session (would have used ~30 calls and produces no different signal
   than the production build that already succeeded). Recommend Adeel runs
   `npm run preview` to spot-check the UI before merging.
5. **Auto punch-out cron** (auto-close lingering open sessions past 11pm)
   NOT shipped. Without it, a staff member who forgets to punch out keeps
   showing "Working Now" until their next punch-in overwrites. Suggest as
   a follow-up n8n workflow if needed.
