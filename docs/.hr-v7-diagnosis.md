# HR V7 — Diagnosis (investigate-first)

**Branch:** `feature/hr-v7-edit-course-review-fixes` (off main `5562841`). **Read-only investigation, 2026-05-30.** Tenants: zate `ac308ab6…`, cosmique `933967dd…`.

## 1A — Employee Edit "Save Changes" fails (PRIORITY) — ROOT CAUSE FOUND
The V6 dialog sends `department_name`, which is **NOT a column** on `hr_employees` (real columns: `department` text + `department_id`). A direct UPDATE with the dialog's field set **fails: `column "department_name" does not exist`** (verified via rolled-back UPDATE). The save also routes through the **n8n webhook `/hr/employee/update`** (`useHR.updateEmployee` → `callWebhookOrThrow`), and that workflow (`U9Yb2JSa46NlOKXC`) shows **no executions** — the webhook path is unreliable amid the queue backlog (Bug #96). The `onError` shows a **generic "Failed to update employee"** (hides the real cause).
**Two compounding causes:** (a) bad column `department_name` in the payload; (b) flaky n8n-webhook round-trip.
**FIX:** switch `updateEmployee` to a **direct supabase update** with only real columns (`first_name,last_name,company_email,phone,position,employment_type,salary,employment_status,department,department_id`), `.eq('id').eq('tenant_id')`, best-effort audit row to `hr_audit_logs`, and surface `error.message` in the toast.

## 1B — Courses: JSON leak + no edit/delete/manual-content
- **JSON leak source:** the catalog card renders `{program.provider && <span>{program.provider}</span>}` as a vendor name. For AI courses `provider` is the JSON content blob → **raw JSON shown** (e.g. "Patient Confidentiality", "LIKE HANDLING THE PATIENTS"). Seed courses store a vendor string there (e.g. "Berlitz UAE") so they look fine. **Fix:** never render raw `provider`; show category/AI-badge instead, and only show provider when it's a plain (non-JSON) string.
- **Storage:** AI content (script/slides/questions/`avatar_video_url`) lives in `hr_training_programs.provider` (JSON). Columns available for edit: `name, description, type, duration_hours, cost, max_participants, status` (+ provider JSON).
- **Missing CRUD:** create/generate/avatar-video work; **edit, delete, manual video/doc** missing; empty courses show a dead-end "No AI-generated content yet".
**FIX:** Edit dialog (name/description/duration/category → update row, `.eq tenant_id`); Delete w/ confirmation (course name) → soft-delete `status='cancelled'` (preserve history); "Add Video URL"/"Add Document URL" → merge into the `provider` JSON additively; empty-course CTA card (Generate AI Content / Add Manually); clean description helper (never print JSON).

## 1C — Reviews: blank recipient names + no edit
`hr_performance_reviews` columns: `employee_id` (recipient), `reviewer_id` (auth-user id), `review_type`, `status`, `comments`, … **No `employee_name`/`period` columns** → V6 Team Reviews renders `review.employee_name` = blank, My Reviews tab is a hard-coded stub. The recipient name resolves cleanly by joining `employee_id → hr_employees` (verified: "Ahmed Al Mansoori", "James Mitchell", …).
**FIX:** in `usePerformance`, attach `employee_name` to each review via an id→name map from `hr_employees`; render "Feedback/Review for [name] · [type] · [date]" with status chips; wire **My Reviews** (reviews where current user is the reviewee, via `useCurrentEmployee`) and **Team Reviews** (all, with names); add **Edit review** (prefilled dialog → update row, `.eq tenant_id`); show the submitted review immediately with the recipient name.
