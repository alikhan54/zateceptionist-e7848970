## PROPOSED BUILD — phased (reuse > rebuild)

### P1 — Outreach engine *(the core missing piece)* — **Medium**
- **New:** 1 workflow `420 HR Recruitment Outreach v1.0` (schedule or webhook). On `hr_job_applications` where screening advanced (or a chosen stage) + candidate is contactable → Claude personalizes per channel → **enqueue** `multichannel_outreach` (SMS/WhatsApp) + email via `/webhook/manual-email/*` → **write `hr_recruitment_outreach`** + set `hr_job_applications.outreach_status='sent'`.
- **Reused (NOT built):** the 3 `multichannel_outreach` drainers, Manual Email Engine, tenant_config creds, the empty-but-ready `hr_recruitment_outreach` log.
- **Tables:** read `hr_job_applications`/`hr_candidates`; write `hr_recruitment_outreach` + `multichannel_outreach` (insert) + `hr_job_applications.outreach_*`.
- **Risks:** only contactable candidates (5/10) reachable until P2; cosmique blocked until P4; reply-handling needs the Reply Router (`JT4ZXxMjIa6SRbla`) wired to advance the candidate.

### P2 — Candidate contact enrichment — **Medium**
- Enrich the apify-sourced candidates (LinkedIn-only) → email/phone. **Reuse** `/webhook/guess-email` (`DpYJu4wwVLUFHImh`) + the existing Phase-3 Apify Profile Scraper (`PWb5cPBpK4FTgwwW`). Map `hr_candidates.enriched_data` → `email`/`phone` columns.
- **Risk:** enrichment hit-rate; verification before sending (avoid bounces / Bug #128).

### P3 — Interview auto-dial — **Medium**
- **Wire the trigger** (the only gap): on a positive outreach reply / candidate-accept (P1 reply flow) → `POST /webhook/hr/ai-interview/start-call` (`bcaK7Lxd0HgtVfqW`). VAPI calls → end-of-call hits `/webhook/hr/ai-interview/call-ended` (`0VEBSpO63nEiR1xh`) → transcript + `recording_url` + scores land in `hr_ai_interviews`. Both webhooks already built — **no sender to build.**
- **Depends on:** P2 (candidate phone) + P4 (tenant VAPI creds). For zate this works today.
- **Risk:** the loop has never executed end-to-end for any tenant — needs one real validated call before trusting.

### P4 — Per-tenant channel provisioning — **Quick (config) / needs-user-action**
- zate ready. **cosmique needs vapi/whatsapp/twilio/smtp credentials** provisioned (via `/webhook/provision-comms` — note the duplicate workflow) or routing through a shared account. Also confirm zate's `smtp_pass` isn't the stale Bug-#128 credential.

### P5 — Funnel + outreach-status UI — **Medium**
- Surface `hr_pipeline_summary` as the funnel; add a "Contact candidate" action + outreach-status badge on the candidate row/dialog (`Recruitment.tsx:~2171`); make sourcing status honest (sent/replied/advanced counts). Frontend-only, additive.

**Build order:** P4(zate-ready)→P1→P2→P3→P5. The senders, logs, VAPI loop, and funnel view all already exist — **the genuinely new code is one outreach orchestrator (P1) + one auto-dial trigger (P3) + UI (P5).**
