- **Generator:** reads course `content_script` → splits into chapters (explicit numbered/heading markers if present, else **sentence-grouped contiguous sections mapped to the ordered slide titles** — content_script here was flowing prose with no headings) → for each: insert chapter (queued) → submit HeyGen `v2/video/generate` with **`callback_id = chapter_id`** → set `status='generating'`, `heygen_video_id`. **No in-workflow poll → returns immediately.** Idempotent (skips chapters already generating/ready). Accepts `max_chapters` (test used 2).
- **Receiver:** HeyGen POSTs `avatar_video.success/fail` → matches by `callback_id`(=chapter id) or `video_id` → sets `video_url` + `status='ready'/'failed'`.
- **Poll-once:** UI "Refresh" / test fallback — checks `generating` chapters vs HeyGen `v1/video_status.get` → updates. **No cron** (sidesteps T19 dark-cron).
- **HeyGen webhook endpoint registered** → `https://webhooks.zatesystems.com/webhook/hr/training/heygen-callback`, events `avatar_video.success/fail`, `endpoint_id a5dbb2b2…`, status enabled. (Signature `whsec_…` not yet verified server-side — hardening item.)
