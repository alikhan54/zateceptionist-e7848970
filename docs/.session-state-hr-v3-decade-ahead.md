## Bugs Found + Fixed (DO NOT reintroduce)

- `useInbox.ts` + `useHR.ts` previously passed `tenantConfig.id` (UUID) to `.eq('tenant_id')` on TEXT-slug tables. Always check Section 2 of CLAUDE.md before filtering by `tenant_id`.
- Phase 4 had 3 silent CHECK constraint violations (`full_name` is a GENERATED column, `match_score` is `numeric(5,4)` ŌåÆ 0-9.9999 not 0-100, `status`/`source` enum limits). Errors were swallowed into `error_log`. Fixed to normalize score `/10`, drop full_name, use `status='active'`, `source` from 17-enum.
- `AIAssistant.tsx` checked `result.data` but `useHRAI.sendMessage` returns the unwrapped body. Fixed to walk `result.data || result`.
- `review_type` CHECK enum is `'self'|'manager'` (rater type), NOT period. Frontend was sending `'quarterly'` ŌåÆ 400. Fixed to `'manager'` + auto-resolved `cycle_id`.
- Training Generator wrote full JSON to `description`, which the card renders as title ŌåÆ raw JSON in UI. Fixed: `description` = 200-char summary, `provider` = full JSON blob.
- `hr_document_acknowledgments` was auto-migrated by Supabase to the `n8n` schema (search_path default of the postgres user used by the pooler). Always run `SET search_path TO public` or fully qualify table names in DDL.
- Auto-Mode RLS used `auth.jwt()->>'user_metadata'->>'tenant_id'` which doesn't match how this Supabase auth session exposes tenant. Switched to `get_user_tenant_uuid()` pattern that all other HR tables use.
- Lovable bundle staleness is common (queue can be 15+ min). Always poll the bundle hash before testing the UI; backend (REST + n8n) can be tested immediately.
