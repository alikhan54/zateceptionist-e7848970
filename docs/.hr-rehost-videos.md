# HR Training Videos — Permanent Re-host (HeyGen → Supabase Storage)

**Branch:** `feature/hr-rehost-videos` (off `feature/hr-custom-avatar` → off `feature/hr-chaptered-videos`)
**Date:** 2026-06-03 · **Tenant tested:** zateceptionist (UUID `ac308ab6-…`)

## Problem
Chaptered training videos are rendered by HeyGen, which returns a **temporary signed URL**
(`https://files2.heygen.ai/...?Expires=<unix>&Signature=...`). That URL **404s once it expires**
(observed `Expires=1781102902`) — unacceptable for a client-facing training feature. Both
completion paths stored that temp URL directly into `hr_course_chapters.video_url`.

## Fix — re-host on completion
On completion, **download the video from HeyGen → upload to permanent Supabase Storage → store the
permanent public URL** in `video_url`; keep the original HeyGen URL in a new `heygen_temp_url` column.

### Storage (additive infra)
- **Bucket:** `training-videos` — **public** (browser playback; a private bucket would force signed
  URLs that *also* expire, defeating the goal). Same model as the existing public `media` bucket.
- **Path (tenant-isolated):** `training-videos/{tenant_id}/{chapter_id}.mp4`
- **Permanent URL:** `https://fncfbywkemsxwuiowxxe.supabase.co/storage/v1/object/public/training-videos/{tenant_id}/{chapter_id}.mp4` (no `Expires`)
- **Column:** `ALTER TABLE hr_course_chapters ADD COLUMN IF NOT EXISTS heygen_temp_url text` (additive)

### `rehost()` behaviour
1. **Idempotent:** if `video_url` already contains `/training-videos/`, skip.
2. Download bytes from the HeyGen URL (`this.helpers.httpRequest({encoding:'arraybuffer'})`).
3. **Upload via raw Node `https.request` + `req.write(buf)`** (see gotcha below).
4. Set `video_url` = permanent Supabase URL; `heygen_temp_url` = original; `error_message` = null.
5. **Graceful failure (`rehost_failed`):** on any download/upload error, keep the chapter, store the
   HeyGen URL in `video_url` as a short-term fallback, and set `error_message='rehost_failed: …'`.
   ⚠ Those chapters still carry the expiry problem until a future retry pass re-hosts them.

## Binary-handling gotchas (both cost a test cycle — documented so they're not repeated)
1. **n8n JSON-stringifies a Buffer body.** Passing `body:<Buffer>` to `this.helpers.httpRequest`
   (even with `json:false`) uploaded `{"type":"Buffer","data":[…]}` — a 6.35 MB JSON blob instead of
   the 1.79 MB MP4. **Fix:** bypass n8n's body handling — use Node `https.request` and `req.write(buf)`.
2. **`URL` is not defined in the n8n Code sandbox.** `new URL(...)` throws `URL is not defined`.
   **Fix:** hardcode the (fixed) Supabase hostname + path instead of parsing.
`require('https')` works because `NODE_FUNCTION_ALLOW_BUILTIN=*`.

## Notes / follow-ups
- The frontend player is **unchanged** — it plays `video_url` regardless of origin; only the data changed.
- **Pre-existing chapters** rendered before this patch still have `files2.heygen.ai` URLs (Leadership ×3,
  Cybersecurity ×2). A one-off backfill (reset to `generating` + trigger the poll, or a small re-host
  sweep) would make them permanent too — not done here to keep the change scoped.
- Inline service-role key in the workflow Code nodes follows the established platform pattern (T23).
