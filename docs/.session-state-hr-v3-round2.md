## What R2 changed

### MyHR Profile editable (frontend + schema)

- **Schema:** `ALTER TABLE hr_employees ADD COLUMN IF NOT EXISTS linkedin_url TEXT, bio TEXT, marital_status TEXT` — applied.
- **Bug fix:** `MyHR.tsx` was querying `hr_employees.auth_user_id` which doesn't

### Claude routing in question gen workflow

- Workflow `K46ve7frdaHYBpn2` rewritten. New nodes:
  - `QG.2 - Build Prompt` looks up `tenant_config` (`subscription_tier`,
    `has_anthropic`, `anthropic_api_key`, `industry`, `subscription_status`)
    via REST. Sets `routing = 'claude'` when:
    - `subscription_status === 'active'`
    - AND `subscription_tier === 'premium'`
    - AND `has_anthropic === true`
    - AND `anthropic_api_key.length > 30`
    Otherwise `routing = 'ollama'`.
  - `QG.3R - Claude or Ollama?` IF node routes traffic.
  - `QG.4c - Claude Generate` HTTP node → `api.anthropic.com/v1/messages`
    with per-tenant `x-api-key` header. Model: `claude-sonnet-4-5-20250929`.
  - `QG.4o - Ollama Generate` keeps existing Ollama path for free tier.
  - `QG.5` parser updated to handle both response shapes
    (Claude: `content[0].text`; Ollama: `message.content`).
- Prompt injects `INDUSTRY CONTEXT: <industry>` line — Cosmique gets
  healthcare-flavored questions, Zate gets technology, etc.
- `target_job_id` param added — when present, QG.5 PATCHes the requisition
  row with the generated questions. This is the hook for auto-mode (deferred
  — see below).

### Auto Punch-Out (schema + admin UI + cron)

- **Schema:** `CREATE TABLE hr_attendance_rules` — one row per tenant
  (UNIQUE on `tenant_id`). RLS enabled with permissive policy at app level
  (tenant scoping enforced by `tenant_id = tenantConfig.id` filter in
  client + service-role server access from cron).
- **UI:** `AttendanceRulesCard.tsx` — admin/manager only (returns null for
  staff). Sections:
  - Auto Punch-Out (toggle + time + assumed-hours + note)
  - Work Hours (expected/overtime/min-break)
  - Punctuality (grace period + late notify)
  - Geofencing (lat/lng/radius)
- **Tab in /hr/attendance:** new "Rules" tab next to List/Calendar/Analytics.
- **n8n cron:** `5WLVNYiiAHJYKuFW '420 HR Auto Punch-Out Cron v1.0'` — active.
  Every 30 minutes:
  1. Read all rows from `hr_attendance_rules` where `auto_punch_out_enabled=true`.
  2. Join with `tenant_config.timezone` for each tenant.
  3. Compute local time in tenant's timezone (`Asia/Dubai`, `UTC`, etc.).
  4. If past cutoff, find open `hr_attendance` rows for today and PATCH them
     with `check_out_time = check_in_time + assume_hours`, set
     `notes = auto_punch_out_notes`, `source='auto'`, compute work_hours.
